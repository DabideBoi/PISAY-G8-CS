Introduction to JavaScript Frameworks

JavaScript frameworks are pre-written JavaScript code libraries that provide developers with pre-built components and functions for common web development tasks. They help streamline development by providing structure, organization, and solutions to common challenges.

Why Use Frameworks?

  • Efficiency: Reduce development time by using pre-built components and patterns
  • Structure: Provide architectural patterns and organization for your code
  • Maintainability: Make code easier to maintain and scale
  • Community: Access to large communities, documentation, and resources
  • Best Practices: Encourage following established best practices

React

React is a JavaScript library for building user interfaces, particularly single-page applications. It's maintained by Facebook and a community of developers.

Key Features

  • Component-Based: Build encapsulated components that manage their own state
  • Virtual DOM: Efficiently updates the DOM by only rendering changed components
  • JSX: Syntax extension that allows writing HTML-like code in JavaScript
  • Unidirectional Data Flow: Data flows down from parent to child components
  • React Native: Framework for building native mobile applications

Basic Example

// Function Component
import React from 'react';

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Class Component
import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

// Using Hooks (React 16.8+)
import React, { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  useEffect(() => {
    document.title = `You clicked ${count} times`;
  }, [count]);
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

Vue.js

Vue.js is a progressive JavaScript framework for building user interfaces. It's designed to be incrementally adoptable and can be used for a wide range of applications.

Key Features

  • Approachable: Easy to understand and integrate with projects
  • Versatile: Can be used as a library or a full-featured framework
  • Reactive: Automatically updates the DOM when data changes
  • Component-Based: Build encapsulated, reusable components
  • Single-File Components: Combine template, logic, and styles in one file

Basic Example

// Vue 3 Example
<template>
  <div>
    <h1>{{ message }}</h1>
    <button @click="increment">Count: {{ count }}</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello Vue!',
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
</script>

// Vue 3 Composition API
<script setup>
import { ref } from 'vue'

const message = ref('Hello Vue!')
const count = ref(0)

function increment() {
  count.value++
}
</script>

Angular

Angular is a platform and framework for building single-page client applications using HTML and TypeScript. It's maintained by Google and provides a comprehensive solution for front-end development.

Key Features

  • TypeScript: Built with and encourages the use of TypeScript
  • Two-way Data Binding: Synchronizes data between model and view
  • Dependency Injection: Built-in system for component dependencies
  • Comprehensive: Complete solution with routing, forms, HTTP client, etc.
  • Testing: Built with testability in mind

Basic Example

// Component
import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <h1>{{ title }}</h1>
      <p>Count: {{ count }}</p>
      <button (click)="increment()">Increment</button>
    </div>
  `
})
export class CounterComponent {
  title = 'Counter App';
  count = 0;
  
  increment() {
    this.count++;
  }
}

// Service
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private data = [];
  
  getData() {
    return this.data;
  }
  
  addData(item) {
    this.data.push(item);
  }
}

Svelte

Svelte is a radical new approach to building user interfaces. It shifts the work from runtime to compile time, resulting in highly optimized vanilla JavaScript.

Key Features

  • No Virtual DOM: Compiles components into efficient imperative code
  • Less Code: Requires less boilerplate than other frameworks
  • Truly Reactive: Fine-grained reactivity without a virtual DOM
  • Small Bundle Size: Generates highly optimized JavaScript
  • Built-in Transitions: Easy animation and transition effects

Basic Example

<script>
  let count = 0;
  
  function increment() {
    count += 1;
  }
</script>

<style>
  button {
    background: #ff3e00;
    color: white;
    border: none;
    padding: 8px 12px;
    border-radius: 4px;
  }
</style>

<h1>Counter</h1>
<p>Count: {count}</p>
<button on:click={increment}>Increment</button>

State Management

As applications grow, managing state becomes more complex. Various state management solutions exist for different frameworks:

Redux (React)

// Action Types
const INCREMENT = 'INCREMENT';
const DECREMENT = 'DECREMENT';

// Action Creators
function increment() {
  return { type: INCREMENT };
}

function decrement() {
  return { type: DECREMENT };
}

// Reducer
function counterReducer(state = { count: 0 }, action) {
  switch (action.type) {
    case INCREMENT:
      return { count: state.count + 1 };
    case DECREMENT:
      return { count: state.count - 1 };
    default:
      return state;
  }
}

// Store
import { createStore } from 'redux';
const store = createStore(counterReducer);

// Using with React
import { Provider, useSelector, useDispatch } from 'react-redux';

function Counter() {
  const count = useSelector(state => state.count);
  const dispatch = useDispatch();
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => dispatch(increment())}>+</button>
      <button onClick={() => dispatch(decrement())}>-</button>
    </div>
  );
}

Vuex (Vue)

// Store
import { createStore } from 'vuex';

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    },
    decrement(state) {
      state.count--
    }
  },
  actions: {
    increment(context) {
      context.commit('increment')
    },
    decrement(context) {
      context.commit('decrement')
    }
  },
  getters: {
    doubleCount(state) {
      return state.count * 2
    }
  }
});

// Using in a component
export default {
  computed: {
    count() {
      return this.$store.state.count
    },
    doubleCount() {
      return this.$store.getters.doubleCount
    }
  },
  methods: {
    increment() {
      this.$store.dispatch('increment')
    },
    decrement() {
      this.$store.dispatch('decrement')
    }
  }
}

Best Practices

  • Component Structure: Keep components small, focused, and reusable
  • State Management: Use appropriate state management for your application size
  • Performance Optimization: Implement code splitting, lazy loading, and memoization
  • Testing: Write unit tests for components and integration tests for workflows
  • Documentation: Document components, props, and expected behaviors
  • Accessibility: Ensure your application is accessible to all users
  • SEO: Consider server-side rendering or static site generation for better SEO
